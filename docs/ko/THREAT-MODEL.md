# 위협 모델

**Language:** [English](../en/THREAT-MODEL.md) · 한국어

> **[보관됨]** 이 프로젝트는 deprecated이며 세션 라우팅은 Orca로 옮겨갔습니다. 되돌리기: [TEARDOWN.md](TEARDOWN.md). 참고용으로 남겨둡니다.

라우터가 무엇을 보호하고 무엇을 보호하지 않는지, 그리고 남아 있는 위험을 정리한다. 라우터가 하는 일은
활성 계정, 도구, 상태 루트, 네트워크 노출을 눈에 보이게, 그리고 서로 분리된 상태로 유지하는 것이다. 벤더가
제공하지 않는 보안을 라우터가 새로 더해 주지는 않는다. 업무 계정을 맡기기 전에 이 문서를 먼저 읽어라.

## 자산

- 자격 증명: Codex `auth.json`, Claude Keychain 항목, API 키, OAuth 토큰.
- 로컬 상태: 설정, 대화 기록, 세션 트랜스크립트, 프로젝트 메모리, 플러그인.
- 코드와 데이터: 각 워크스페이스에 들어 있는 저장소와 파일.
- 신원: 각 표면에서 어떤 계정(개인 또는 회사)이 활성 상태인지.
- 네트워크 도달 범위: 원격 클라이언트가 조작할 수 있는 로컬 컨트롤 플레인(Codex `app-server`, Claude Code Remote Control, MCP 서버, 로컬 웹 UI).

## 신뢰 경계

- 계정별 상태 루트(`CLAUDE_CONFIG_DIR`, `CODEX_HOME`)가 디스크에서 개인과 회사를 분리한다.
- macOS Keychain은 config 디렉터리별로 Claude 로그인을 분리한다(이 머신에서 확인함, 버전에 따라 다름).
- 테일넷은 사설 경계다. Tailscale Funnel은 이 경계를 뚫고 공개 인터넷으로 나간다.
- 벤더 서버는 라우터가 접근할 수 없는 웹·모바일 계정 상태를 보관한다.

## 위협과 완화책

### T1. 자격 증명 유출
토큰이나 키가 로그, 트랜스크립트, git, 터미널 출력에 남는 경우.
- 완화책: 라우터는 `auth.json`, `.credentials.json`, 토큰, 키, Keychain 시크릿을 절대 출력하거나 복사하지 않는다. `ai doctor`와 `ai keychain`은 존재 여부, 파일 모드, 그리고 되돌릴 수 없는 지문만 읽는다. `.gitignore`는 모든 계정 루트와 로그를 제외한다.
- 잔여 위험: 사용자 본인의 명령은 여전히 유출될 수 있다(`cat auth.json` 하지 말 것). Codex `auth.json`은 모드 600의 평문이므로, 디스크 읽기 권한이 있는 사람은 누구나 볼 수 있다.

### T2. 잘못된 계정 사용
개인 계정으로 업무 작업을 돌리거나, 회사 코드를 개인 계정으로 보내는 경우.
- 완화책: `ai <tool> <workspace>`는 계정을 워크스페이스 기준으로 기본 설정한다. `--account`로는 의도적으로 섞을 수 있고, 개인 워크스페이스에 회사 계정이 붙는 불일치가 생기면 경고한다. `ai resolve`는 실행 전에 정확한 계정과 env를 미리 보여 준다.
- 잔여 위험: 경고는 실행을 막지 않는다. `ai doctor`는 어떤 Keychain 항목이 존재하는지 보여 줄 뿐, 어떤 계정이 활성 상태인지는 아니다. 세션 안에서 `/status`로 확인하라.

### T3. 메모리·기록의 교차 오염
한 계정의 Claude Code 메모리나 Codex 기록이 다른 계정으로 새어 드는 경우.
- 완화책: 상태 루트를 분리하면 `history.jsonl`, 트랜스크립트, 메모리가 계정별로 유지된다.
- 잔여 위험: Claude Code 자동 메모리는 git 저장소 단위로 키가 매겨진다. 그래서 두 계정이 같은 저장소에서 작업하면 그 저장소의 메모리를 공유한다. `autoMemoryDirectory`나 `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1`로 분리하라. 루트 사이에서 `auth.json`을 복사하지 마라. Codex refresh 토큰은 일회용이라, 복사본은 곧 만료된다.

### T4. 개인/업무의 우발적 혼용
개인 저장소에 있으면서 회사 계정을 띄우거나, 그 반대인 경우.
- 완화책: 워크스페이스 축이 파일과 로그를 고정한다. 워크스페이스 안에서 실행하면 그 자리에 머물고, 밖에서 실행하면 경고한 뒤 워크스페이스 루트로 옮겨 준다. 로그는 워크스페이스 아래에 쌓이므로, 세션의 출처가 디스크에 남는다.
- 잔여 위험: `router.env`의 워크스페이스-경로 매핑이 잘못돼 있어도 이를 막는 장치는 없다.

### T5. 로컬 에이전트 컨트롤 플레인 노출
Codex `app-server`, Claude Code Remote Control, MCP 서버, 로컬 웹 UI가 네트워크에서 접근 가능해지는 경우.
- 완화책: 라우터는 인바운드 포트를 절대 열지 않는다. 원격 접근은 Tailscale Serve(사설, ACL로 통제) 또는 SSH로 문서화되어 있다. Funnel(공개 인터넷)은 옵트인이며 고위험으로 표시된다. Codex `app-server` WebSocket 모드는 실험적이므로, 인증 없이 노출하지 마라.
- 잔여 위험: Claude Code Remote Control은 인바운드 포트 없이 아웃바운드 HTTPS를 쓰지만(위험이 더 낮다), Claude.ai 로그인 정보를 가진 사람은 누구나 그 세션을 조작할 수 있다. Funnel은 한번 켜면 공개 인터넷이다.

### T6. 산출물·세션 출처 소실
어떤 계정이나 워크스페이스가 산출물, 트랜스크립트, 패치를 만들었는지 알 수 없는 경우.
- 완화책: 트랜스크립트는 `<workspace>/.ai-logs/<tool>/<account>-account/` 아래에 기록되므로, 경로가 도구, 계정, 워크스페이스를 남긴다. `ai logs`로 목록을 볼 수 있다.
- 잔여 위험: 웹·모바일 산출물(ChatGPT 캔버스, Claude Artifacts)은 서버 쪽에 있고 라우터는 이를 캡처하지 않는다. 벤더 UI에서 내보내라.

### T7. 모바일 원격 제어의 모호함
폰에서 로컬 세션을 조작하는데, 그것이 어떤 로컬 계정이나 호스트를 제어하는지 알 수 없는 경우.
- 완화책: Codex Remote는 호스트가 같은 ChatGPT 계정과 워크스페이스에 로그인되어 있어야 한다. Claude Code Remote Control은 특정 로컬 세션 하나를 조작한다. `ai doctor`와 `ai remote doctor`가 호스트 상태를 보고한다.
- 잔여 위험: 폰이 어떤 세션에 붙을지는 라우터가 아니라 벤더 앱이 정한다. 원격 제어를 끊으려면 로그아웃하라.

## 라우터가 보호하지 않는 것

- 벤더 서버 쪽 계정 상태(웹·모바일 로그인, Projects, Artifacts).
- 악성코드나 다른 로컬 사용자가 디스크나 Keychain을 읽는 경우.
- 잘못된 `router.env` 매핑.
- 사용자가 직접 붙여넣거나 파이프로 넘긴 것.

표면별 격리는 [SURFACES.md](SURFACES.md), 지원 매트릭스는 [SUPPORT.md](SUPPORT.md)를 참고하라.
